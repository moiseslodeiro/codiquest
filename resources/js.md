Nivel 1 - Info

JavaScript es un lenguaje de programación que le da vida a las páginas web. Imagina que una página web es como una hoja de papel, estática y plana. JavaScript es como la magia que hace que esta hoja cobre vida: permite hacer cosas como botones interactivos que responden cuando haces clic, formularios que validan tus respuestas antes de enviarlas y animaciones que hacen que las imágenes se muevan. En resumen, JavaScript es el "pegamento" que une diferentes partes de una página web y la convierte en algo dinámico y emocionante.

Las páginas web dinámicas necesitan almacenar información para mostrar o guardar, para ellos javascript (y el resto de lenguajes), disponen de, entre otras cosas, variables.

Una variable en programación es como una etiqueta que pegas en una caja para recordar lo que hay adentro. Puedes guardar diferentes tipos de cosas en estas cajas, como números, palabras o pedazos más grandes de información. Cuando quieras usar lo que guardaste, simplemente miras la etiqueta en la caja y sabes qué está adentro. Así, las variables permiten que el programa recuerde y manipule datos de manera más fácil y organizada.

Nivel 3 - Order

El nombre de una variable es la palabra o conjunto de letras (sin espacios) que eliges para identificar y referirte a esa variable en tu código de programación. Es como el apodo que le das a una caja para recordar lo que hay dentro. 

let $city$

Fallo: Recuerda, ¡no pueden contener espacios!

----

Nivel 4 - Info

Si el nombre de la variable está compuesta por más de una palabra, haremos uso de la nomenclatura camelCase. CamelCase es una convención de escritura de palabras en la que las palabras se unen sin espacios y cada palabra después de la primera comienza con una letra mayúscula. Esto se usa comúnmente en nombres de variables y funciones en programación para hacer que los nombres sean más legibles y distintivos. Por ejemplo, "miVariableCamel" es un ejemplo de CamelCase donde las palabras "mi", "Variable" y "Camel" están conectadas, y cada palabra comienza con una letra mayúscula, excepto la primera. ¡Recuerda, sin espacios!

----

Nivel 5 - Choose

Es importante elegir nombres descriptivos y significativos para que puedas entender fácilmente qué representa esa variable cuando leas el código más tarde. Los nombres de variables suelen comenzar con una letra o un guión bajo, seguidos de letras, números o guiones bajos adicionales.

myCity
mc
mC
